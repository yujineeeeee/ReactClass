리액트 프로젝트 구성

node_modules : node.js 모듈이 설치되는 폴더, 리액트 프로젝트가 동작하는데 필요한 모듈이 설치되는 폴더
    - bulid 시 필요 없는 부분은 자동으로 제외됨
    - 일반적으로 사용자는 신경 쓸 일이 없음

public : 리액트 프로젝트의 기본 파일이 위치하는 곳, index.html 파일에 있는 div 태그를 리액트 프로그램이 조작하여 화면을 구성함

src : 리액트 프로젝트의 소스코드가 존재하는 폴더

index.js : 리액트 프로그램의 시작점, 리액트 프로젝트 시작 시 index.html 파일 안의 div 태그 중 id 값이 'root' 인 태그를 검색하여 index.js 를 로딩하고 index.js의 내용을 실행하여 화면을 최종 구성

MPA : Multi Page Application 의 줄임말, 웹 페이지가 여러개 있는 형태의 웹 사이트, 기존 방식의 웹 사이트

SPA : Single Page Application 의 줄임말, 웹 페이지가 단 하나로 이루어진 웹 사이트, 자바스크립트를 이용하여 필요에 따라서 웹 페이지를 새로 그려내는 웹 사이트

Virtual DOM(Document Object Model) : 리액트에서 웹 페이지를 그려내기 위해서 사용하는 방식,
    - 기본적으로 웹은 화면을 한번 출력하고 다시 수정된 화면을 출력 시 모든 화면을 새로 그려야 함
    - 수정된 화면을 출력하기 위해서 새로 화면을 그리면 화면 깜빡임 현상이 발생함
    - 조금만 수정이 발생해도 모든 리소스를 다시 호출하기 때문에 성능이 떨어지게 됨
    - Virtual DOM 을 사용하여 원본 화면을 그대로 두고, 화면에서 Virtual DOM 을 이용하여 화면을 수정하고 마지막에 원본 화면인 DOM 과 Virtual DOM 을 교체함

컴포넌트 - 리액트 화면 구성 요소, 화면의 각 부분을 잘게 쪼개어 화면을 만들고, 만들어진 화면을 재사용하여 전체 화면을 구성하는 방식, 자바스크립트로 구성되어 있음

JSX : Javascript XML 의 줄임말, 화면을 구성하기 위한 형태를 html 태그로 사용하고 중간에 js 코드가 들어가는 형태로 구성해 놓음, JSX 의 속성명은 html 과 조금 차이가 남


프리티어 설정 :
{
  "useTabs": false, // 탭 사용 시 공백문자로 변환
  "printWidth": 100, // 가로 너비 100 칸으로 설정
  "tabWidth": 2, // 탭 크기 공백 2칸으로 설정
  "trailingComma": "all", // 나열 항목 사용 시 마지막에 늘 쉼표 사용
  "semi": true, // 실행 줄 마지막에 항상 세미콜론 붙임
  "singleQuote": true // 따옴표를 작은 따옴표로 변경
}
